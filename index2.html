The Problem
The Why
Today
Every Prod Review ticket requires a human to manually open the Amplify link in the correct mini app, click through every flow in the AC, then repeat as Building Guard, Tenant Admin, and Visitor Registration Host to check permissions. Then visually diff production against the design prototype.
The Back-and-Forth
When something's off, the ticket gets kicked back to engineering. Engineering fixes it, pushes it back. Design reviews again. Another round trip — sometimes two or three before it's approved. This cycle eats hours from both teams every single sprint.
Edge Cases That Slip Through
A persona sees a button they shouldn't. A status pill is missing its dot prefix. A table has pagination instead of infinite scroll. Date formatting is wrong. These things make it to production because no human can catch every detail across 20+ roles and a growing surface area.
Bug Ledger
Does the first pass automatically. Picks up the ticket when it hits Prod Review, runs every check, and posts a pass/fail report back to Linear with screenshots and a session recording. Humans still approve — they just skip the grunt work.

Validation Prompt Categories
Text Casing
Title Case for nav, breadcrumbs, product names · Sentence case for buttons, labels, card titles, table headers
Button Verbs
"Add [Item]" for existing entities · "Create [Item]" for new ones · every button checked
Status Pills & Tables
Dot prefix required on all status text (• Active) · Infinite scroll only, no pagination controls
Date / Time & Placeholders
MM/DD/YYYY · No leading zeros on time · Search inputs: "Search by {criterion}"
Copy Tone & Empty States
Address as "You / Your" · Present tense · Missing data → "-" · Not applicable → "N/A"

Under the Hood
Tech Stack
AI Agent
Claude
Reads the ticket, browses the app via Playwright, finds bugs, and writes the structured QA report — fully autonomously.
Browser
Playwright
Headless browser the agent controls to click, screenshot, and record a full session video of every test run.
Ticket Source + Reporting
Linear
Tickets trigger the agent. QA reports, screenshots, and session videos land back here automatically — no human copy-paste.
Memory
LanceDB + HuggingFace
Local vector store so the agent learns across runs — selectors, known bug patterns, conventions — and gets sharper over time.
Orchestration
GitHub Actions
Glue that runs the whole pipeline on every PR — credentials, artifact publishing, and report posting to Linear.

If We Had More Time
What's Next
Robustness
Context Window Handling
Right now, when the agent hits its context window limit mid-run, it drops the remaining work on the floor — silently. We'd build proper checkpoint-and-resume logic so long runs can recover gracefully instead of producing incomplete reports.
Coverage
Full Persona Expansion
We currently test against three key personas. We'd expand to cover all roles across the HqO platform — giving every user type its own agent instance that runs permission checks, scoped data validation, and role-specific AC steps in parallel.
